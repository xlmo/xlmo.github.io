Mac下编译Emacs

获取源码

  • bzr branch http://bzr.savannah.gnu.org/r/emacs/trunk
  • git clone git://git.savannah.gnu.org/emacs.git
    这个可能有点慢。（Emacs源码8xxM）

进入目录

./autogen.sh
./configure  —with-ns
make
make install

注意这里一定要先执行一下autogen.sh，不然直接./configure的话会报错（找不到install-sh）

此方法Linux下也能成。

文件权限与目录配置

文件类型

  • [d] 目录
  • [-] 文件
  • [l] 连接文件
  • [b] 设备文件里的可供存储的接口设备
  • [c] 设备文件里的串行端口设备
  • [s] 套接口文件

  • [p] 管道 pipe
    改变文件属性与权限

  • chgrp 改变文件所属用户组

  • chown 改变文件所有者（包括用户组）
  • chmod 改变文件的权限
    目录权限有[x]才可进入此目录，有[w]则可删除目录下文件，不论其文件属性或属主为何。这条对root用户不适用，root用户即使目录没有[x]也能进入。

FHS

FHS只定义了三个目录定义：/、/usr和/var。

  • / 根目录，与开机有关
  • /usr Unix Software Resource的所写，放置可分享与不可变动的数据

  • /var 常态性变动的文件
    因根目录与开机有关，所以以下目录不能放到与根目录不同的分区中：

  • /etc 配置文件

  • /bin 重要执行文件
  • /dev 所需的设备文件
  • /lib 执行文件所需的函数库与内核所需的模块
  • /sbin 重要的系统执行文件

HTML5特性检测

检测是否支持HTML5的某种特性，原则：

  1. 检测全局对象。
  2. 创建一个元素，检测该元素的DOM对象是否拥有特定的属性。
  3. 创建一个元素，检测该元素的DOM对象是否拥有特定的方法，同时调用这个方法并检查它的返回值。
  4. 创建一个元素，给这个元素的DOM对象设定特定的属性值，然后检测浏览器是否保留了这个属性值。
     

Canvas

return !!document.createElement(‘canvas’).getContext;

Canvas Text

//在检测支持Canvas的基础上来检测Canvas Text
var o_c = document.createElement(‘canvas’);
var context = o_c.getContext(‘2d’);
return typeof context。fillText ＝＝ ‘function’；

Video

return !!document.createElement(‘video’).canPlayType;

Video Formats

//在检测支持Video的基础上来分别检测是否支持以下编码格式
var v = document.createElement(‘video’);

/ 编码格式1,返回值：
“probably” 可以播放
“maybe” 可能可以播放
“” 无法播放 /
return v.canPlayType(‘video/mp4; codecs=”avc1.42E01E, mp4a.40.2”‘);

/ 编码格式2 /
return v.canPlayType(‘video/ogg; codecs=”theora,vorbis”‘);

/ 编码格式3 /
return v.canPlayType(‘video/webm; codecs=”vp8, vorbis”‘);
Local Storage

return (‘localStorage’ in window) && window[‘localStorage’] != null;

Web Workers

//双重否定强制返回布尔值
return !!window.Worker;

Offline Web Applications

return !!window.applicationCache;

Geolocation

return !!navigator.geolocation;

Input Types

//input元素默认类型为text，将其设置为要检测的类型，然后查看是否仍然是text.
var i = document.createElement(‘input’);
i.setAttribute(‘type’,’attribute’);
return i.type !== “text”;

Placeholder Text

var i = document.createElement(‘input’);
return ‘placeholder’ in i;

Form Autofocus

var i = document.createElement(‘input’);
return ‘autofocus’ in i;

Microdata

return !!document.getItems;

最后，可以使用 modernizr 来检测浏览器是否支持某些HTML5特性。

配置本地svn开发环境

建立版本库

sudo mkdir /var/svn/
创建版本仓库 
sudo svnadmin create /var/svn/repos
修改配置,注意配置项目的前面不能有空格。 
sudo vim /var/svn/repos/conf/svnserve.conf
anon-access = read
auth-access = read
password-db = passwd
auth-db = authz
修改用户配置 
sudo vim /var/svn/repos/conf/passwd
username = password
修改权限配置 
sudo vim /var/svn/repos/conf/authz
[groups]
user_groupname = username1,username2
@user_groupname = 权限
或者
username = 权限
启动subversion 
svnserve -d -r /var/svn/repos/
结束服务 
killall -9 svnserve

免签名联机调试

首先设备要越狱，然后安装AppSync。注意我的设备是4.2.1的，而网上给的源地址里面已经没有AppSync for 4.2之类的了，4.0以上都用AppSync for 4.0+代替了。

进入目录

cd /Developer/Platforms/iPhoneOS.platform/Developer/SDKs/iPhoneOS4.2.sdk
sudo vi SDKSettings.plist

将CODE_SIGNING_REQUIRED的值改为NO。

修改项目的属性。在Build这一tab栏中 找到Code Signing Identity 这项 将其设置为Don‘t Code Sign.设置”Set Active SDK”为Device。

Done。

Smarty3的autoload问题

今天一项目中定义了一个__autoload(),结果怎么都不管用，排查后发现是Smarty的问题。

这个项目使用的是Smarty3，它注册了自己的autoload:

spl_autoload_register('smartyAutoload');
造成__autoload()无效了，解决方法很简单，就是下面这行： 
spl_autoload_register('__autoload');

MagickWand的怪异问题

出现了一个很诡异的情况，如下代码中:

$magick_wand=NewMagickWand();
MagickReadImage($magick_wand,’rose.jpg’);

NewMagickWand这个函数是存在的，并且phpinfo中也显示了有magickwand项，但是这里的$magick_wand既不是对象也不是资源……造成了读取图片时出现参数错误的问题。

问题已解决！是因为magickwand和gmagick有冲突，去掉扩展gmagick.so就可以了。真是怪异！
